# DiplomProject

Этот проект содержит набор автотестов для проверки функциональности личного кабинета Ростелеком ID. Цель — обеспечить стабильность интерфейса и защитить пользователей от несанкционированного доступа.

Что проверяют тесты
Тесты охватывают ключевые сценарии авторизации, регистрации и восстановления пароля.
Цель — убедиться, что интерфейс работает корректно и защищает конфиденциальные данные.

Функционал, который проверяется:
Авторизация:  

Вход по телефону и email.  
Авторизация с неверным паролем.  
Переключение между способами входа.
Регистрация:  

Создание нового аккаунта с уникальным email.  
Попытка зарегистрировать уже использованный email и телефон.  
Проверка валидации имени и фамилии.
Восстановление пароля:  

Выбор способа доставки кода (SMS или email).  
Ввод короткого и длинного пароля.  
Проверка сложности пароля.  
Несовпадение паролей при подтверждении.

Вход по временному коду — это особый сценарий, который объединяет функциональность восстановления пароля и авторизации. Тесты проверяют, что система корректно генерирует и отправляет одноразовый код, а также позволяет войти в аккаунт с его помощью.

Требования для запуска
Перед запуском тестов убедитесь, что установлены:

Python 3.x (рекомендуется последняя стабильная версия).  
PyTest — менеджер для запуска тестов.  
Selenium — библиотека для автоматизации браузеров.  
Chromedriver — драйвер для управления браузером Chrome.

 Запуск тестов
Базовый запуск
Запустить все тесты в каталоге tests/ можно командой: pytest tests/

Запуск отдельного файла
Чтобы запустить только тесты из файла test_login.py: pytest tests/test_login.py
Чтобы запустить только тесты из файла test_recovery.py: pytest tests/test_recovery.py
Чтобы запустить только тесты из файла test_registration.py: pytest tests/test_registration.py
Чтобы запустить только тесты из файла test_temporary_code.py: pytest tests/test_temporary_code.py

Подробный отчёт
Для просмотра полной трассировки ошибок: pytest tests/ -v

Генерация HTML-отчёта
Для наглядного отчёта выполните: pytest --html=report.html
